Responsive Layout Approach

The current game uses fixed-width tables side by side (Queen + King). How should it adapt to smaller screens?

A

Stack on Mobile

Queen and King tables stack vertically on screens < 768px. Tabs stay horizontal but scroll. Worker/Army tables also stack. Minimal CSS changes using media queries on the existing layout.

Pros

  • Least disruption to existing layout
  • All content visible, just reordered
  • Works with current table-based structure

Cons

  • Long scrolling on mobile
  • Tables still wide on small screens
B

Tab-Based Mobile View

On mobile, add sub-tabs: "Queen" / "King" instead of showing both side by side. Worker types also get sub-tabs. Reduces scroll depth significantly.

Pros

  • Much less scrolling
  • Focused view per section
  • Better mobile UX

Cons

  • More JS logic for sub-tab switching
  • Harder to compare Queen vs King at a glance
C

Responsive Tables with Horizontal Scroll

Keep the current layout but wrap each table in a horizontal scroll container on mobile. Stack Queen/King vertically. Tables keep full width but are swipeable.

Pros

  • No data hidden or reorganized
  • Pure CSS solution
  • Familiar pattern for data-heavy apps

Cons

  • Horizontal scroll can feel clunky
  • Not the most polished mobile experience
